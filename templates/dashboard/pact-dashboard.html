How PACT Tracking Works

PACT (Programmatic Agent Constraint Toolkit) instruments Claude Code's hook system to observe governance activity in real time. No code runs inside the AI model — everything happens at the shell layer.

Event Sources

  • Edit events — PostToolUse hook on Edit/Write. Fires after every file change, logging path and session ID.
  • Preflight checks — PostToolUse hook runs cognitive redirections (defined in preflight-checks.yaml) against each edit. Session-scoped dedup prevents repeat fires.
  • Hook blocks — PreToolUse guards that reject edits violating rules (no print(), no Hive, no hardcoded colors, etc.).
  • Flow reads — Logged when a feature flow doc is read before editing critical system files.
  • Governance — Updates to SYSTEM_MAP.yaml, schema exports, provider cheatsheets.
  • Commits — Captured by PreToolUse on git commit commands.
  • Session start — SessionStart hook registers the agent model and unique session ID.

Data Flow

Each hook calls pact-event-logger.sh which appends a JSON line to .claude/pact-events.jsonl. The PACT server (pact-server.py, port 7246) reads this file every second and serves new events via /events?after=N. This dashboard polls that endpoint.

Session Identity

Each session gets a unique ID at startup (timestamp + random hex). This ID is written to a temp file and read by all downstream hooks, ensuring every event is attributed to the correct session — even when Claude and Gemini sessions overlap.

What's NOT Tracked

Token usage and cost are not available in hook input. File reads (without edits), web searches, and internal model reasoning are not captured. Only actions that touch the governance system are instrumented.

Dashboard Startup Preference

Controls whether Claude asks to start the dashboard at the beginning of each session. Stored in ~/.claude/pact-config.json.
